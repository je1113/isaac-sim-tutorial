# Module 08 — 캡스톤: 나만의 파이프라인 조립

**권장 소요: 통합 프로젝트 (1~2주, 자율 배분)**

---

## 1. 학습 목표

- 모듈 1~7에서 다룬 요소(USD/Python API, PhysX 튜닝, 로봇/매니퓰레이션, 센서, 합성 데이터, Isaac Lab)를 하나의 일관된 프로젝트로 통합할 수 있다.
- 프로젝트를 남이 이해할 수 있는 형태(블로그 글)로 정리할 수 있다.

## 2. 선수 지식 확인

- 모듈 1~7의 체크포인트를 모두 통과했는지 먼저 점검. 특히 모듈 2(Python API)와 모듈 3(PhysX)의 이해가 얕으면 이번 모듈에서 반드시 발목을 잡는다.

---

## 3. 프로젝트 방향 선택 (택 1, 혹은 자유 조합)

이번 과정은 정답이 정해진 실습이 아니라 지금까지 쌓은 요소를 조합해 스스로 문제를 정의하는 단계다. 아래는 예시일 뿐, 자기 관심사에 맞게 자유롭게 재조합해도 된다.

**A. 인식-조작 통합 파이프라인**
모듈 4(Franka pick-and-place) + 모듈 5(카메라 인식) + 모듈 6(합성 데이터)을 이어붙여: 합성 데이터로 간단한 물체 인식(색상/모양 분류 정도의 가벼운 모델도 무방)을 만들고, 그 인식 결과로 실제 pick-and-place를 수행하는 엔드투엔드 데모.

**B. 커스텀 RL 태스크 확장**
모듈 7의 커스텀 태스크를 더 어렵게 확장 (장애물 추가, 목표를 매 에피소드 무작위 지점으로, 여러 하위 태스크 조합 등) 하고, Event Manager 랜덤화로 강건성을 검증하는 프로젝트.

**C. 진단/튜닝 도구화**
모듈 3에서 만든 진단 스크립트들을 일반화해, 임의의 USD 로봇을 입력받아 "떨림/통과/무게중심" 등을 자동으로 스캔해 리포트를 출력하는 작은 CLI 도구로 발전.

## 4. 진행 가이드

1. **목표 정의 (반나절)** — 무엇을 만들 것인지, 성공 기준이 무엇인지(예: "10회 시도 중 8회 이상 pick 성공") 먼저 명확히 적는다.
2. **모듈별 자산 재사용 계획 (반나절)** — 어느 모듈의 어느 스크립트/에셋을 그대로 가져다 쓸지 매핑.
3. **구현 (본체)** — 반복적으로 실행/디버깅. 막히는 지점이 이전 모듈에서 이미 겪은 문제 유형(PhysX 불안정, 컨테이너 Xform 함정, VRAM 부족 등)인지 먼저 대조.
4. **정량/정성 평가** — 목표 기준에 맞춰 성공률/학습 곡선/시각 결과를 정리.
5. **영상 + 블로그 원고 작성** — 과정 전체(성공만이 아니라 실패와 삽질 포함)를 기록. 이전 과정의 "알려진 문제와 해결" 표 형식을 그대로 재사용 권장.

## 5. 체크포인트 & 산출물

**체크포인트**: 목표로 정의한 성공 기준을 실제로 달성했거나(성공), 왜 달성하지 못했는지를 원인 수준까지 분석했다(실패해도 분석이 명확하면 통과).

**최종 산출물**:
- 데모 영상 (성공 케이스 + 흥미로운 실패 케이스)
- 코드 (`practicum/m08_capstone/`)
- 블로그 원고 초안: 문제 정의 → 시도 → 실패와 원인 → 최종 결과 → 배운 점
